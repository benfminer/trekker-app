## 1. Project Overview

TRACE Trekkers is a year-long collective fitness challenge for the entire TRACE school community — students, teachers, and staff — pooling their miles and steps to walk around the world, starting and ending in San Diego, CA (~24,900 miles total). The app replaces a Google Form + spreadsheet system with a purpose-built experience: a clean, frictionless log entry page, a bold live map showing the group's collective progress, and a competitive site-based leaderboard that pits TRACE's four campuses against each other to see who can trek the most. A small admin team manages submissions through a protected dashboard.

## 2. Goals & Success Metrics

- **Primary goal:** Keep the whole school community engaged in the initiative consistently throughout the year — not just at the start
- **Secondary goals:**
  - Make collective progress visible and exciting in real time
  - Fuel friendly inter-site competition to sustain engagement
  - Reduce friction for submitting miles/steps
  - Celebrate milestones (countries, continents, oceans, capitals) as they're reached
- **How success will be measured:** Sustained submission activity throughout the school year; community engagement with the map and leaderboard; completion of the circumnavigation route

## 3. Target Users

The entire TRACE school community — adult students, teachers, and staff at TRACE's four campuses in San Diego, CA. Users are tech and phone-savvy. They submit individually or as a group/class. No accounts or login required — anyone with the link can participate. The experience should feel empowering, celebratory, and competitive in a good-natured way.

## 4. Core Features — v1 Scope

### Must Have
- Open-access submission form (no login) — log miles or steps, individually or as a group/class, with optional site selection
- Automatic step-to-mile conversion at 2,500 steps per mile
- Live map showing the great circle route from San Diego around the world, with a marker at the current collective position
- Milestone system — triggered when the group enters a new country, crosses a sea or ocean, enters a new continent, or reaches a major city or capital
- Milestone celebration UI — something visually rewarding when a milestone is hit
- **Site-based leaderboard** — public page showing cumulative miles per campus (Trace North, Trace South, Trace East, Trace West), animated dashboard with live rankings
- Migration of all historical data (~7,135 miles) from previous Google Form/CSV exports — historical data has no site attribution
- Admin interface to review, edit, and delete submissions
- **Admin: change own password** (while logged in)
- **Admin: create new admin accounts** from the dashboard
- **Admin: forgot password email flow** (transactional email service TBD)
- Rotating inspirational quotes on the public-facing pages
- Outlook-safe HTML announcement email for launch

### Out of Scope (v1)
- User accounts or login
- Personal activity logs or individual progress tracking
- Site attribution for historical CSV data
- 3D Mapbox globe (Sprint 3 stretch goal only)

## 5. User Journey

1. User opens the app on their phone or computer — no login required
2. They see the live map with the current position of the group on the route, plus rotating inspirational quotes
3. They tap/click "Log Activity" to go to the submission page
4. They enter their name or class name, the date, choose miles or steps (app converts automatically), and optionally select their site from a dropdown
5. They submit — confirmation is shown
6. The map updates to reflect the new total; if they tagged a site, the leaderboard updates too
7. If a milestone is crossed, a celebration moment triggers on the map

**Leaderboard flow:**
1. User navigates to the Leaderboard page from the main nav
2. Sees four animated site cards ranked by total miles
3. Rankings re-sort with motion as new submissions come in (or on page load)

**Admin flow:**
1. Admin navigates to a protected admin route (link in bottom nav, small and unobtrusive)
2. Logs in with email + password
3. Views all submissions in a table — can edit, delete, or flag entries
4. Can change their own password from account settings
5. Can create additional admin accounts
6. If password forgotten: requests reset email, follows link, sets new password

## 6. Design & UX Direction

### Visual Personality
Two distinct experiences within one app:

- **Log entry page:** Clean, warm, grounded. Easy to use on a phone. Feels like a field journal — light background, earthy tones, tactile. No distractions. Fast.
- **Live map page:** Bold, adventurous, expedition-like. The map is the hero. Feels alive and in-motion.
- **Leaderboard page:** Full-bleed dark dashboard. Four site cards with animated mile counters. Live ranking that re-sorts with motion. Bold progress bars per site. Sports scoreboard meets mission control.

The overall aesthetic is: expedition journal meets adventure map. Light backgrounds with warm earth tones as the foundation. Orange and black are accent colors — used for calls to action, highlights, and moments of energy — not the dominant palette. Think worn canvas, aged paper, trail markers, topographic maps.

### Key UI Moments
- The map view with the route and current position marker — this is the heart of the app
- Milestone celebration — a satisfying moment when the group crosses into a new country, continent, or body of water
- The submission confirmation — should feel like a contribution, not just a form receipt
- Rotating inspirational quotes — appear subtly on map and log pages
- Leaderboard ranking re-sort animation — the moment one site overtakes another should feel electric

### Color Palette
- **Base:** Warm off-white / light parchment (e.g. #FAF7F2 or similar)
- **Surface:** Slightly warmer white for cards and inputs (e.g. #FFFFFF or #FDF9F4)
- **Earth tones (primary):** Warm tans, muted khakis, olive greens, dusty browns — used for backgrounds, borders, and supporting text
- **Orange (#FF6B00 or similar):** Accent — CTAs, active states, highlights, success moments
- **Black / near-black:** Accent — headings, key labels, map elements
- **Muted warm gray:** Body text, secondary labels
- **Leaderboard exception:** Full dark palette (#1a1a1a or similar) for the leaderboard page — this page deliberately breaks the warm parchment aesthetic to feel like a live broadcast dashboard

### Navigation
- **Public nav:** Map, Log Activity, Leaderboard — prominent, top-level
- **Admin nav:** Small, bottom of page — unobtrusive, not discoverable unless you know it's there

### Design References
- Field journal / expedition logbook aesthetic
- Topographic map and trail guide visual language
- National park signage — bold but warm, never corporate
- School colors (orange + black) as accent, not base
- Clean form design: single-column, large tap targets, minimal labels
- Sports scoreboard / live broadcast dashboard for the leaderboard

Mobile-first. The log entry page should be thumb-friendly. The map should scale gracefully to desktop where it can be more immersive. The leaderboard should work on mobile but shine on a large screen — consider it being displayed on a TV in each campus.

### Accessibility

## 10. Content & Data Model

**Core data:**
- Submissions: name/class name, date, input type (miles or steps), input value, converted miles, site (optional: Trace North / Trace South / Trace East / Trace West / null), imported flag, flagged flag, timestamp
- Milestones: type (country, continent, ocean, city/capital), name, mile marker, triggered (boolean), triggered date
- Route: pre-computed great circle path from San Diego around the world (~24,900 miles), stored as a series of coordinates
- Admin users: email, password digest, active flag
- Admin sessions: token digest, expires_at
- Password reset tokens: admin_user_id, token digest, expires_at, used flag

**Data flow:**
- Users submit via open form → stored in PostgreSQL (site field optional, defaults to null)
- Total miles calculated from all submissions (including null-site) → used to compute map position
- Site-tagged miles only feed the leaderboard
- Position checked against milestone markers → milestone triggered if threshold crossed
- Admin can view, edit, delete any submission

**Historical data:**
- ~7,135 miles from previous Google Form CSVs — imported with site = null
- Support for additional CSV imports in the future

**Leaderboard data:**
- Aggregated SUM of `converted_miles` grouped by `site` WHERE `site IS NOT NULL`
- Four fixed sites: Trace North, Trace South, Trace East, Trace West
