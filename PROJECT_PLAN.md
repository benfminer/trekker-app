## Project Summary

TRACE Trekkers is a collective mileage-tracking web app for the TRACE school community. Students and staff log miles via an open (no-login) form; a small admin team approves submissions through a protected dashboard; and the cumulative school-wide total displays live on an animated Mapbox globe. The app must launch April 1, 2026 — two weeks away — with ~7,135 miles of historical CSV data migrated and visible on day one.

---

## Phases & Milestones

### Phase 1: Foundation — Target: 2026-03-22
**Goal:** Rails API scaffolded, database schema live, CSV migration script written and tested, React app initialized with routing and Tailwind config.

**Exit criteria:** `rails db:migrate` runs clean; CSV import script loads all historical records without errors; React app boots and hits a health-check endpoint.

### Phase 2: Core Features — Target: 2026-03-28
**Goal:** All MVP features working end-to-end: submission form, admin dashboard (auth + approve/reject/edit), live map with cumulative route, progress counter.

**Exit criteria:** A logged submission moves from pending to approved and appears on the map without manual intervention beyond admin approval. Admin can log in, review a batch, and approve in under 5 minutes.

### Phase 3: Polish, QA & Launch — Target: 2026-04-01
**Goal:** Mobile responsiveness verified, edge cases handled, CSV data confirmed in production, app deployed to AWS.

**Exit criteria:** All MVP features pass QA on mobile and desktop. Historical data visible on map. Submission-to-map flow tested end-to-end in production.

## Risks & Mitigations

| Risk | Likelihood | Impact | Mitigation |
|------|------------|--------|------------|
| Mapbox globe animation eats the week | High | High | **DECIDED 2026-03-19:** Ship animated polyline on 2D map for launch. 3D globe is a stretch goal for Sprint 3 only if time allows. |
| CSV column structure is messy or inconsistent | Med | High | Audit CSV immediately (task 1.5). Build import script defensively with row-level error logging. |
| Deploy issues eat launch day | Med | High | Deploy to staging by March 28 — never leave deploy for April 1. |
| Milestone geodata sourcing takes too long | Med | Med | Use a pre-built country/capital dataset (Natural Earth or similar). Timebox to half a day. |
| Open form gets spammed | Low | Med | Calculated risk for MVP. Admin can delete bad submissions. Add CAPTCHA post-launch if needed. |

## Scope Triage

If time becomes tight, cut in this order:

1. **First to cut:** Animated Mapbox route — ship a static polyline instead
2. **Second to cut:** Milestone celebrations UI — show milestones as a list, no animation
3. **Third to cut:** Milestone geodata seeding — launch with manual milestone entry later
4. **Non-negotiable:** Log entry form, admin edit/delete dashboard, live map (any form), historical CSV migration
